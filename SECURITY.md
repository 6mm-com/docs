# Security Policy

Please do not report security vulnerabilities through public GitHub issues.

If you believe you have found a vulnerability in 6MM documentation, APIs, SDKs, authentication, webhook handling, or trading entry flows, contact the 6MM team through the official support or security reporting channels listed in the documentation:

- https://docs.6mm.com/resources/support/security-reporting
- https://docs.6mm.com/resources/contact/support

When reporting a security issue, include:

- Affected product area or endpoint
- Environment and time range
- Reproduction steps
- Logs, request IDs, screenshots, or other evidence
- Whether the issue is currently active
