6MM Agentic AI Hub is the AI-native access layer for 6MM markets, account workflows, and future controlled trading operations.

It is designed for users who want to use 6MM from tools such as Claude Desktop, Cursor, Codex CLI, ChatGPT Desktop, VS Code, Gemini CLI, Windsurf, and other AI clients that can call local tools, remote MCP servers, or command-line programs.

<h2 id="what-ai-hub-provides">What AI Hub provides</h2>

<CardGroup cols={2}>
<Card title="6MM CLI" icon="fa-duotone fa-terminal" href="/ai-hub/cli">
Run 6MM market and account commands from a terminal, automation script, or coding agent.
</Card>
<Card title="Local MCP" icon="fa-duotone fa-computer" href="/ai-hub/local-mcp">
Run a local MCP server so AI clients can call 6MM tools while API keys remain on the user's machine.

<Card title="Agentic Skills" icon="fa-duotone fa-layer-group" href="/ai-hub/agentic-skills">

<h2 id="capability-model">Capability model</h2>

| Layer | Best for | First supported scope |
| --- | --- | --- |
| Hosted MCP | Zero-install public data access | Public market data and trading rules |
| Local MCP | Full AI client integration | Public data first, then account-read and controlled trading |
| CLI | Coding agents, scripts, terminals | Public data first, then account-read and preview |
| Remote MCP | Institution-managed infrastructure | Self-hosted MCP endpoint controlled by the user or institution |
| Agentic Skills | Repeatable AI workflows | Templates and guardrails before executable trading workflows |

<h2 id="recommended-starting-path">Recommended starting path</h2>

1. Start with Hosted MCP if you only need public market data.
2. Use Local MCP when an AI client needs account context or future trading actions.
3. Use CLI when your AI agent can execute shell commands or you want scriptable JSON output.
4. Use Remote MCP when an institution needs a long-running MCP endpoint under its own infrastructure.
5. Add Agentic Skills once the team has clear workflows and risk boundaries.

<h2 id="current-product-stance">Current product stance</h2>

AI Hub should be treated as an access layer over 6MM APIs and SDKs. It does not replace the security model, signing model, or account-level permissions described in the Developers and SDK sections.

The initial public scope should stay read-only and market-data focused. Account access, order preview, and controlled trading should be enabled only after permission scopes, risk checks, explicit confirmation, and audit logging are in place.
