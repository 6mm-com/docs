6MM 支持服务面向授权合作方、进行中的接入项目和生产运营。除另有约定外，终端用户支持仍由合作方平台负责。

<h2 id="choose-the-right-path">选择合适路径</h2>

| 场景 | 推荐路径 | 建议提供 |
| --- | --- | --- |
| 新合作或方案评估 | 合作接入路径 | 产品类型、目标地区、上线时间、期望接入模式。 |
| SDK、API、Webhook 或环境问题 | 技术支持路径 | 接口、SDK 版本、requestId、复现步骤、日志。 |
| 生产事件或交易链路异常 | 事件升级路径 | 影响范围、开始时间、受影响用户、订单 ID、当前缓解措施。 |
| 素材、文案或上线资料调整 | 运营支持路径 | 页面、语言、修改内容、上线日期、确认负责人。 |
| 漏洞或疑似滥用行为 | 安全上报路径 | 技术细节、受影响接口、证据、紧急程度、问题是否仍在发生。 |

<h2 id="severity-guide">优先级参考</h2>

| 优先级 | 示例 |
| --- | --- |
| 紧急 | 生产交易、鉴权、Token 签发或资金相关链路不可用。 |
| 高 | 主要合作方流程异常，但存在临时绕行方案。 |
| 普通 | 接入咨询、配置请求、文档问题或计划性上线支持。 |

<h2 id="before-contacting-support">联系支持前</h2>

- 确认受影响的产品范围和环境。
- 收集 requestId、订单 ID、时间戳、用户映射 ID、Webhook eventId 和日志。
- 生产问题建议使用支持请求模板。
