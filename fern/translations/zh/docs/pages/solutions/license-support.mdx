牌照规划是整体上线计划的一部分。合适路径取决于目标市场、用户类型、产品范围、资产托管模型以及各方运营责任。

<h2 id="planning-inputs">规划输入</h2>

| 输入 | 为什么重要 |
| --- | --- |
| 目标司法辖区 | 决定监管要求和本地专业审查重点。 |
| 用户类型 | 零售、专业、机构或合作方分发用户可能需要不同控制措施。 |
| 产品范围 | 现货、合约、钱包、托管和经纪业务会影响合规模型。 |
| 运营模式 | 白标、嵌入式、经纪商和机构模式具有不同职责边界。 |

<h2 id="how-to-use-this-documentation">如何使用本文档</h2>

这些文档用于在法律审查前对齐技术和运营要求，帮助团队描述架构、数据流、签名、安全、资产流转和交易生命周期职责。

> **重要**
>
> 牌照相关决策必须由合作方结合目标司法辖区咨询具备资质的法律与合规顾问。
