嵌入式交易通常是接入到已有产品中，而不是作为一个孤立站点独立上线。最合适的接入路径，取决于用户原本在哪里使用产品，以及哪个系统负责身份、资产和运营。

<h2 id="common-scenarios">常见场景</h2>

| 场景 | 建议起点 | 说明 |
| --- | --- | --- |
| App 或钱包希望增加交易入口 | Trading Widget SDK | 通过原生或 H5 入口，在用户授权后打开内嵌交易体验。 |
| 经纪商或渠道平台希望提供合约交易 | Agent SDK + Trading Widget SDK | 客户关系、后端校验和用户准入仍保留在合作方系统。 |
| 平台希望做定制化交易流程 | API + 指定 SDK 模块 | 使用后端 API 处理账户、划转和订单流程，只暴露必要前端入口。 |
| 现有交易所希望补充交易市场 | 流动性 + API 接入 | 组合市场深度、账户流程和运营监控能力。 |

<h2 id="product-decisions-before-integration">接入前需要确认</h2>

- 用户从首页卡片、资产页、活动页还是账户中心进入交易。
- 哪个系统负责登录态、用户准入和地区可用性判断。
- 资产划转、结算记录和用户工单如何展示。
- 哪些事件需要进入内部数据分析、CRM 或风控运营系统。
