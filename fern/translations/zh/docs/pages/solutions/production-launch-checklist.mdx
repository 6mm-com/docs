在向生产用户开放 6MM 交易能力前，可以用这份清单做最后检查。它适合产品、工程、运营、风控和客服团队一起评审。

<h2 id="launch-checklist">上线检查</h2>

| 领域 | 检查项 |
| --- | --- |
| 产品 | 入口、用户文案、风险提示、语言覆盖和客服路径已准备。 |
| 后端 | API 凭证、签名、token 签发、Webhook、重试和日志已实现。 |
| 交易 | 订单、划转、仓位、保证金、费用和清算行为已评审。 |
| 安全 | 密钥、访问角色、监控、事件响应和审计日志已准备。 |
| 运营 | 客服流程、升级路径、对账和日常监控已明确负责人。 |

<h2 id="recommended-sequence">推荐顺序</h2>

1. 完成沙盒或内部环境验证。
2. 跑通登录、入口、划转、交易和客服的端到端用户路径。
3. 确认监控和事件响应负责人。
4. 审核公开文案、本地化素材和资源下载。
5. 先进行可控窗口上线，再扩大推广。

<h2 id="after-launch">上线后</h2>

初始上线阶段建议每日复盘。入口转化、token 签发失败、订单错误、价差质量、划转完成率、Webhook 重试和客服工单需要放在一起观察。
