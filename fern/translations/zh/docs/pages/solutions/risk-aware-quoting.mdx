做市不只是挂单。在真实交易环境中，报价需要响应波动率、仓位敞口、对冲条件和运营限制。

<h2 id="risk-aware-liquidity-goals">风险感知流动性目标</h2>

- 保持市场可交易，同时避免产生不可控敞口。
- 当波动率、深度或对冲条件变化时调整报价行为。
- 将流动性运营与保证金、清算和账户风险规则协调起来。
- 支持合作方增长，同时减少自建大型做市团队的压力。

<h2 id="areas-to-align-before-launch">上线前需要对齐</h2>

| 领域 | 需要决定 |
| --- | --- |
| 支持市场 | 哪些交易对和结算资产优先上线。 |
| 用户画像 | 零售、机构、渠道还是混合用户流。 |
| 监控指标 | 每日需要查看哪些深度、价差、成交和风险指标。 |
| 升级机制 | 异常波动、结算延迟或影响用户的事件由谁处理。 |
