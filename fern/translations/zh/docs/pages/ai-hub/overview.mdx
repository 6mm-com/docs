6MM Agentic AI Hub 是面向 AI 客户端和 AI Agent 的 6MM 接入层，用于读取 6MM 市场数据、处理账户工作流，并在后续阶段支持受控交易操作。

用户不需要在交易页面、API 文档和脚本之间来回切换，可以直接在 AI 客户端里询问 6MM 行情、解释市场状态、检查账户风险，或生成订单预览。AI 客户端不会替代 6MM API 的安全模型；它通过 CLI、MCP 或工作流模板调用结构化的 6MM 工具。

<h2 id="what-you-can-ask-an-ai-client">用户可以怎么问</h2>

AI Hub 适合处理这类问题：

```text
查看 BTC-USDT-PERP 的盘口，并解释当前流动性。
```

```text
对比 BTC 和 ETH 过去 24 小时的资金费率。
```

```text
查询我的测试网仓位，并解释清算风险。不要下单。
```

```text
预览一笔小额 BTC 多单，并解释费用、保证金影响和风险检查。
```

<h2 id="why-ai-agents-need-a-dedicated-access-layer">为什么 AI Agent 需要专门的接入层</h2>

AI Agent 更适合调用结构化工具，而不是读取网页或复制粘贴的长文本。AI Hub 提供：

| 需求 | 6MM 方案 |
| --- | --- |
| 实时市场上下文 | 用于 symbols、盘口、成交、资金费、费用和规则的公开市场工具与资源。 |
| 清晰权限 | 区分公开数据、账户只读、订单预览和受控交易权限范围。 |
| 更安全执行 | 先预览的订单流程、明确确认、幂等键、限额和审计日志。 |
| 多客户端适配 | Hosted MCP、Local MCP、CLI、Remote MCP 和 Agentic Skills 适配不同 AI 环境。 |

<h2 id="setup-paths">接入路径</h2>

| 路径 | 适合场景 | 用户需要知道 |
| --- | --- | --- |
| Hosted MCP | 快速读取公开市场数据 | 公开数据无需安装，也不需要用户 API key。第一阶段不以交易为目标。 |
| Local MCP | AI 客户端需要私有账户上下文 | MCP 服务在本机运行，凭证留在用户本机。 |
| CLI | 编码 Agent、脚本和终端用户 | 输出适合 agent 解析，也可以不依赖 MCP 服务。 |
| Remote MCP | 机构或团队需要自有端点 | 用户或机构控制托管、凭证、网络访问和日志。 |
| Agentic Skills | 可复用工作流 | Skill 引导 AI 使用已授权工具完成分析和风险步骤。 |

<h2 id="what-is-available-first">第一阶段先支持什么</h2>

AI Hub 应先从公开市场数据和交易规则资源开始：

- 市场列表。
- 行情 ticker。
- 盘口。
- 成交。
- K 线。
- 资金费率。
- 系统状态。
- 费用、合约规格、保证金规则和风控规则资源。

私有账户和交易能力应分阶段开放：

- 账户只读工具 需要明确的只读凭证。
- 任何真实下单前都应先完成订单预览。
- 受控交易需要权限范围、服务端风险检查、用户确认、幂等键和审计日志。
