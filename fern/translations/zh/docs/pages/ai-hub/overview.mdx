6MM Agentic AI Hub 是面向 AI 客户端和 AI Agent 工作流的 6MM 接入层，用于让用户在 AI 工具中安全读取 6MM 行情、账户信息，并在后续阶段使用受控交易能力。

它面向 Claude Desktop、Cursor、Codex CLI、ChatGPT Desktop、VS Code、Gemini CLI、Windsurf，以及其他能够调用本地工具、远程 MCP 服务或命令行程序的 AI 客户端。

<h2 id="what-ai-hub-provides">AI Hub 提供什么</h2>

<CardGroup cols={2}>
<Card title="6MM CLI" icon="fa-duotone fa-terminal" href="/zh/ai-hub/cli">
在终端、自动化脚本或 coding agent 中执行 6MM 行情和账户命令。
</Card>
<Card title="Local MCP" icon="fa-duotone fa-computer" href="/zh/ai-hub/local-mcp">
在用户本机运行 MCP server，让 AI 客户端调用 6MM 工具，同时 API key 留在本机。
</Card>
<Card title="Hosted MCP" icon="fa-duotone fa-cloud" href="/zh/ai-hub/hosted-mcp">
使用 6MM 官方托管的公开市场数据工具，无需安装本地服务。
</Card>
<Card title="Agentic Skills" icon="fa-duotone fa-layer-group" href="/zh/ai-hub/agentic-skills">
使用市场分析、组合风险、清算保护和订单规划等工作流模板。
</Card>
</CardGroup>

<h2 id="capability-model">能力模型</h2>

| 层级 | 适合场景 | 第一阶段范围 |
| --- | --- | --- |
| Hosted MCP | 零安装公开数据访问 | 公开行情和交易规则 |
| Local MCP | 完整 AI 客户端接入 | 先公开数据，再账户只读和受控交易 |
| CLI | Coding agent、脚本、终端 | 先公开数据，再账户只读和订单预览 |
| Remote MCP | 机构自管基础设施 | 用户或机构自托管 MCP endpoint |
| Agentic Skills | 可复用 AI 工作流 | 先提供模板和风控边界，再扩展可执行交易工作流 |

<h2 id="recommended-starting-path">推荐开始路径</h2>

1. 只需要公开市场数据时，先用 Hosted MCP。
2. AI 客户端需要账户上下文或未来交易动作时，使用 Local MCP。
3. AI agent 可以执行 shell 命令，或需要 JSON 输出时，使用 CLI。
4. 机构需要长期在线且由自己控制的 endpoint 时，使用 Remote MCP。
5. 团队流程和风险边界明确后，再引入 Agentic Skills。

<h2 id="current-product-stance">当前产品边界</h2>

AI Hub 是 6MM API 和 SDK 之上的 AI 接入层，不替代开发者文档中的安全模型、签名模型和账户权限。

第一阶段应保持只读和公开市场数据优先。账户访问、订单预览和受控交易，需要在权限分层、风险检查、明确确认和审计日志到位后再开放。
