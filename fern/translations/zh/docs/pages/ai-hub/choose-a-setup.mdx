选择接入方式时，先回答两个问题：

1. 你想在哪个 AI 客户端里使用？
2. 你只需要公开数据，还是需要私有账户访问或交易工作流？

如果不确定，公开数据先从 Hosted MCP 开始；私有账户工作流先从 Local MCP 开始。

<h2 id="pick-by-use-case">按使用场景选择</h2>

<CardGroup cols={2}>
<Card title="只需要公开市场数据" icon="fa-duotone fa-cloud" href="/zh/ai-hub/hosted-mcp">
使用 Hosted MCP。无需本地安装，不需要用户 API key，也不访问私有账户。
</Card>
<Card title="希望 AI 客户端读取我的账户" icon="fa-duotone fa-computer" href="/zh/ai-hub/local-mcp">
使用 Local MCP 和只读凭证。建议从 测试网或只读模式开始。
</Card>
<Card title="我的 agent 可以执行终端命令" icon="fa-duotone fa-terminal" href="/zh/ai-hub/cli">
使用 6MM CLI。适合 Codex、Claude Code、Gemini CLI、脚本和 CI 任务。
</Card>
<Card title="团队需要自管端点" icon="fa-duotone fa-server" href="/zh/ai-hub/remote-mcp">
使用 Remote MCP。将 MCP 服务部署在团队自己控制的基础设施中。
</Card>
</CardGroup>

<h2 id="quick-decision-table">快速选择表</h2>

| 目标 | 推荐方式 | 原因 |
| --- | --- | --- |
| 让 AI 客户端查询公开市场数据 | Hosted MCP | 无需本地安装，也不需要用户 API key。 |
| 让 Claude Desktop、Cursor、ChatGPT Desktop 调用 6MM 工具 | Local MCP | AI 通过用户本机 MCP 服务 调用工具。 |
| 让 Codex、Claude Code、Gemini CLI 或脚本调用 6MM | CLI | 命令输出可使用 JSON，方便 Agent 解析。 |
| 给公司内部提供统一 MCP 端点 | Remote MCP | 机构自己控制部署、凭证、访问和日志。 |
| 标准化重复的分析或风控流程 | Agentic Skills | 模板定义工具使用、风控边界和输出格式。 |

| ChatGPT Desktop | 可用 | 推荐 | 可用 | 私有账户工具优先 Local MCP。 |

| Codex CLI | 可用 | 可用 | 推荐 | CLI 通常是最直接路径。 |
| Claude Code CLI | 可用 | 可用 | 推荐 | 根据工作流选择 CLI 或 Local MCP。 |

<h2 id="pick-by-setup-path">按接入路径选择</h2>

| 方式 | 安装方式 | API key 位置 | 第一阶段范围 | 账户与交易范围 |
| --- | --- | --- | --- | --- |
| Hosted MCP | 无需安装 | 公开数据不需要用户 API key | 公开市场数据 | 第一阶段不建议开放 |
| Local MCP | 用户安装本地包 | 用户本机 | 公开数据和账户只读 | 未来交易能力的推荐路径 |
| CLI | 用户安装命令行包 | 用户本机 | 公开数据和脚本 | 适合订单预览和自动化 |
| Remote MCP | 用户自己部署服务 | 用户云或机构云 | 公开数据或账户只读 | 用户控制部署时可逐步开放 |
| Skills | 文档模板或可安装模板 | 取决于底层工具 | 分析工作流 | 必须遵循工具权限 |

<h2 id="security-rule-of-thumb">安全选择原则</h2>

使用能完成任务的最低风险方式：

- 公开市场数据：Hosted MCP 足够。
- 私有账户数据：Local MCP 或用户自管 Remote MCP。
- 交易预览：Local MCP 或 CLI，优先使用测试网。
- 真实交易：必须有明确权限、订单预览、用户确认、限额和审计日志。
